parallel claude code sessions, each in its own git worktree

Ever wish you could have Claude work on multiple things at once, in the same code base? Fix a bug in one session while prototyping a feature in another, or compare different approaches side by side.

Plural lets you run many Claude sessions at once, each in an isolated git branch. Switch freely between tasks. Merge when ready. No conflicts, no context-switching overhead.

what you can do

Branch on options: When Claude offers multiple approaches, fork the session and explore them all in parallel. Compare implementations, pick the best.
Parallel exploration: Try two different approaches to the same problem, compare results, pick the best one.
Bug + feature: Fix a production bug in one session while continuing feature work in another.
Multi-repo work: Coordinate changes across frontend and backend repos simultaneously.
Code review assist: Have Claude explain a PR in one session while you implement feedback in another.
Safe experiments: Let Claude try risky refactors in isolation. Delete the branch if it doesn't work out.

how it works

◇

git worktree isolation

Each session creates a branch and worktree. Claude edits files there, not in your working directory. Merge when you're ready.

◈

independent sessions

Each session has its own Claude process. Send messages to one while another is working. No waiting.

⑂

fork on options

When Claude presents options, press Ctrl+P to fork and explore each approach in parallel. Child sessions inherit full conversation history.

◆

non-blocking permissions

Permission prompts appear inline per-session. Approve when you're ready, or switch to a different session first.

▶

merge or PR

When a session's work is ready, merge directly to main or create a GitHub PR. Auto-commits any pending changes.
