Run multiple Claude sessions on the same codebase—each in its own git branch. When Claude offers different approaches, fork the session and try them all in parallel. Switch freely. Merge the winner.

Requires Claude Code CLI installed and authenticated.

install via homebrew

$ brew tap zhubert/tap && brew install plural

install via nix

$ nix profile install github:zhubert/plural

install via devbox

$ devbox global add github:zhubert/plural

what you can do

Import GitHub issues: Import issues from your repo and have Claude automatically start working on each fix.
Branch on options: When Claude presents numbered options, automatically fork into parallel sessions to explore each approach.
Parallel exploration: Manually fork a session to try different approaches to the same problem. Compare results, pick the best.
Bug + feature: Fix a production bug in one session while continuing feature work in another.
Multi-repo work: Coordinate changes across frontend and backend repos simultaneously.
Code review assist: Have Claude explain a PR in one session while you implement feedback in another.
Safe experiments: Let Claude try risky refactors in isolation. Delete the branch if it doesn't work out.
Review before merge: View all changes in a session before merging or creating a PR.

how it works

◇

git worktree isolation

Each session creates a branch and worktree. Claude edits files there, not in your working directory. Merge when you're ready.

◈

independent processes

Each session runs its own Claude CLI process. Send messages to one while another is working. No waiting.

⑂

conversation history

Forked sessions inherit full conversation history. Child sessions know everything the parent discussed.

◆

non-blocking permissions

Permission prompts appear inline per-session. Approve when you're ready, or switch to a different session first.
